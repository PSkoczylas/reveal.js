Co można robić

Jeżeli chcesz pracować wokół klepania aplikacji to praca w korpo nie jest wcale jedyną możliwością

Praca w korporacji

- W zależności od firmy własne projekty/dla klientów/outsourcing

- Mają pieniądze na szkolenia i uczenie młodych pracowników

- Często sztywna hierarchia

- Często stare projekty i przestarzałe technologie

- Duża ilość pracowników powoduje, że jesteś jedynie pionkiem w wielkim systemie

- Często mogą zapłacić więcej i mają dobry socjal

Praca w Software House

- Praca dla klientów, często bezpośrednio

- Bezpośredni kontakt z szefem firmy

- Możliwość częstej zmiany projektów

- Sporo nowych projektów == kontakt z nowymi technologiami

- Mniej sztywna hierarchia

Praca w Startupie

- Praca przy jednym dużym projekcie (lub podprojektach) == trudniej się wdrożyć

- Możliwość tworzenia ciekawego produktu na wczesnym etapie

- Gdzieniegdzie możliwość dostania opcji na udziały

- Możliwość wdrożenia własnych pomysłów do produktu

Java

- Język obiektowy, kompilowany do kodu wykonywanego przez maszynę wirtualną, statycznie typowany

- Najprawdopodobniej najpopularniejszy język programowania

- Dużo pracy

- Już nie najnowocześniejsza, nowe wersje wprowadzają nowości

- Używana przy aplikacjach mobilnych (android) i typu Enterprise

Android

- Praca zarówno w dużych firmach jak i małych firmach

- Bardzo popularny - dużo osób umie i już nie tak łatwo o pracę

- Możliwość tworzenia własnych aplikacji, które zawojują świat

Aplikacje Enterprise

- Praca głównie w korporacjach

- Dużo pracy, mnóstwo utrzymywanych projektach

- Frameworki: Spring, Hibernate, Java EE

C#/.NET

- Język obiektowy, statycznie typowany należący do microsoftu

- Używany głównie przez korporacje do pisania aplikacji enterprise, webowych (ASP.NET, .NET MVC) i desktopowych (.NET)

- Sporo pracy, promowany przez microsoft

- Składnia podobna do Javy, jednak obecnie to C# szybciej wprowadza nowości

- Praca raczej tylko na systemie Windows

JavaScript

- Język dynamicznie typowany wykorzystywany przez przeglądarki internetowe, dzięki któremu nie trzeba odświeżać strony

- Praca głównie przy front-endzie, mniej przy back-endzie

- Znienawidzony przez wielu, stara się to zmienić przy najnowszych standardach

- Przez znienawidzenie istnieją języki kompilowane do JavaScriptu, np.: TypeScript czy CoffeScript

- Mnóstwo front-endowych frameworków webowych, m.in.: Angular 1, Angular 2, React, Backbone, Ember i podobno codziennie powstają nowe ;)

- Sporo pracy, najwięcej przy frameworkach Angular i React

- Praca zarówno w korporacjach jak i startupach, wszędzie musi być jakiś front-end, wyboru za bardzo nie ma

- Często przydatna jest znajomość frameworków JS przez programistów webowych back-endowych lub full-stacków

- Praca również przy back-endzie dzięki node.js

- Niezły dla startupów, występuje również w korporacjach, wykorzystywany często przy aplikacjach wymagających wielowątkowości

- Głównie praca jako full-stack developer

- Coraz większa popularność

C++

- Język strukturalny i obiektowy, statycznie typowany

- Pracy coraz mniej, jednak ciągle jest

- Praca przy gamedevie, aplikacjach typu embedded

PHP

- Język imperatywny i interpretowany, zbudowany specjalnie do pisania aplikacji internetowych

- Praca przy aplikacjach webowych

- Sporo wykorzystywanych frameworków, m.in.: Laravel, Symfony

- Sporo pracy, głównie w mniejszych firmach, ale i w większych coś się zdarzy

- Uznawany przez wiele osób za nieporozumienie, przez co powoli traci, jednak wciąż powstają w nim nowe startupy więc szybko nie zniknie

Python

- Język obiektowy, imperatywny i w mniejszym stopniu funkcyjny dynamicznie typowany, stawiający na prostotę, często polecany na początek

- Praca przy frameworku DJango, przy pisaniu skryptów systemowych, testowaniu

- Główny język wykorzystywany przy Machine Learningu i Big Data (głównie dzięki bibliotece numpy)

- Używany często jako język pomocniczy przy pisaniu skryptów na linuxa

- Średnio jeśli chodzi o ilość pracy, jednak da się znaleźć

DJango

- Używany z powodzeniem przez wiele startupów

- Za wolny przy dużym obciążeniu

Ruby

- Język obiektowy, dynamicznie typowany

- Podobny do Pythona, stawia na przyjemność z kodzenia i produktywność

- Praca głównie przy frameworku webowym Ruby on Rails, można znaleźć też testowanie i machine learning

Ruby on Rails

- Pozwala na szybkie zmiany i wprowadzanie nowych elementów

- Zasada cośtam ponad konfigurację

- Używany z powodzeniem przez wiele startupów

- Za wolny przy dużym obciążeniu

- Sporo pracy na zachodzie i USA, średnio w Polsce, mało w Lubline, głównie startupy i software house'y

- Dużo pracy zdalnej

Swift

- Używany do pisania na platformy Apple

Nowsze technologie

Scala

- Język funkcyjny działający na maszynie wirtualnej Javy

- Spora popularność i ciągle zyskuje

Elixir

- Język funkcyjny

- Głównie wykorzystywany z frameworkiem Phoenix do aplikacji webowych

- Wykorzystywany przez niektóre startupy/software house, dobrze skalowalny

- Promowany przez społeczność Ruby

Clojure

- Język funkcyjny na platformie wirtualnej Javy

- Głównie wykorzystywany przy aplikacjach webowych

Go

- W zamierzeniu ma zastąpić C++

- Używany przy aplikacjach webowych wymagających dużej wydajności

Rust

F#
